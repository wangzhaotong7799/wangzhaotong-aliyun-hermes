# 🛡️ 守护 (Shielda) — 工作记忆

## 👤 主人信息
- **安全红线**：宝塔面板代码不能动（铁律第7条）
- **系统类型**：医疗数据管理系统，涉及患者信息
- **关注点**：数据隐私、权限控制、操作日志

## 📋 已知风险点
- API 有游客权限控制（guest 角色限制）
- JWT 令牌 + 角色权限验证（admin/assistant/guest）
- 处方记录含患者姓名、电话、地址等敏感信息
- 操作有 StatusChangeLog 记录追踪

## 🧠 质量经验
- 前端输入需要 escapeHtml 转义（已实现）
- 后端 API 有统一的权限拦截
- gunicorn 重启保护已加 --reuse-port
- OOM 风险：内存 1.8G，Swap 6G
